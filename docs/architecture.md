# Architecture Overview: StarBridgeTutoring

**StarBridgeTutoring** is designed as a highly interactive, real-time "Lumio Killer" EdTech SaaS platform. It leverages modern, loosely coupled components to deliver a scalable, mobile-first experience optimized for low-bandwidth environments.

## Core Stack

- **Frontend:** React + Vite
- **Styling:** Tailwind CSS (YELLOW brand theme, responsive UI)
- **State Management:** React Context API + Custom Hooks (`useRealtimeClassroom`)
- **Backend/DB:** Supabase (PostgreSQL, GoTrue for Auth, Realtime via WebSockets, Storage)
- **Hosting:** Vercel (Frontend), Supabase (Database + Auth)

## Data Flow

1. **Authentication:**
   - Teachers, Students, and Admins authenticate via Supabase Auth.
   - Teachers use standard Email/Password or OAuth.
   - Students join using a short `class_code` generated by the Teacher.

2. **Real-Time Data (The Classroom):**
   - The Teacher Dashboard uploads slides (PDF/Images) to **Supabase Storage**.
   - The Teacher's view sends state updates (current slide, quiz triggers) to a **Supabase Realtime Channel**.
   - Student viewers subscribe to the same channel, instantly syncing their browser view to the Teacher's state.

3. **Database Interactions:**
   - **PostgreSQL** handles persistent storage for `classes`, `lessons`, `quizzes`, and student `responses`.
   - Row Level Security (RLS) ensures that students only see data for the class they are joined in.

## Low Bandwidth Optimizations

- **Asset Caching:** Images and slides are served via Supabase's global CDN.
- **Payload Minimization:** Realtime WebSocket messages carry only state diffs (e.g., `current_slide_index: 3`) rather than full assets.
- **Lazy Loading:** React components (like the interactive viewers) are imported dynamically.

## Component Architecture

- **Teacher View (`TeacherDashboard`, `TeacherClassroom`):**
  Has full read/write access to lessons and controls the Realtime broadcast.
- **Student View (`ClassroomStudent`, `JoinClass`):**
  Read-only for lessons/slides; write access for specific endpoints like `responses` (quizzes/polls/shout-it-out).
- **Interactive Modules (`IslamicModules`):**
  A dedicated module for Islamic Studies, cleanly separated from general components.
